03 ‚Äî Build Roadmap (8h/day ¬∑ ~12 Days)
01
Project Setup & Database
Days 1‚Äì2 ¬∑ 16h
  • Create Spring Boot project at start.spring.io ‚Äî add all dependencies
  • Set up PostgreSQL database (local or Docker)
  • Write application.yml ‚Äî DB URL, JPA config, server port 8080
  • Set up Redis + Elasticsearch via docker-compose
  • Create project folder structure: controller / service / repository / model / dto / security
‚úÖ OUTPUT: Spring Boot starts, connects to PostgreSQL, Docker running
02
Authentication System (JWT)
Days 3‚Äì4 ¬∑ 16h
  • Create User model: id, username, email, password, role (USER/ADMIN)
  • Build UserRepository + UserService
  • Write JwtService: generateToken, validateToken, extractUsername
  • Write JwtAuthenticationFilter (runs on every request)
  • Configure SecurityConfig: permit /api/auth/**, protect everything else
  • Create AuthController: POST /api/auth/register and POST /api/auth/login
  • Test with Postman ‚Äî get JWT token back
‚úÖ OUTPUT: Users can register and login, receive JWT token
03
Core Data Models & APIs
Days 5‚Äì7 ¬∑ 24h
  • Create Hotel model: id, name, city, pricePerNight, rooms, rating, imageUrl, type
  • Create Booking model: id, user, hotel, checkIn, checkOut, guests, totalPrice, status
  • Create Cafe / Restaurant model (similar to Hotel)
  • Build all Repositories with JPA
  • Build Services with business logic (HotelService, BookingService)
  • Build Controllers: GET/POST/PUT/DELETE for hotels and bookings
  • Add Redis cache on hotel list: @Cacheable("hotels")
  • Add Elasticsearch HotelSearchRepository for smart search
‚úÖ OUTPUT: Full CRUD APIs for hotels + bookings, search + caching working
04
Expense Calculator
Day 8 ¬∑ 8h
  • Create TripRequest DTO: checkIn, checkOut, hotelId, foodPreference, transport, extras
  • Create TripExpense DTO: totalCost, hotelCost, foodCost, transportCost, extrasCost
  • Write ExpenseCalculatorService: sum all costs automatically
  • Create endpoint: POST /api/calculator/calculate
  • Test with Postman ‚Äî verify total price calculation is correct
‚úÖ OUTPUT: One API call returns full trip cost breakdown in MAD
05
Frontend MVP (Thymeleaf + Tailwind)
Days 9‚Äì10 ¬∑ 16h
  • Create Thymeleaf templates: home.html, hotels.html, booking.html, dashboard.html
  • Add Tailwind CSS (CDN) to templates for styling
  • Build hotel listing page with search + filter
  • Build booking form page
  • Build dashboard with expense calculator form
  • Add Chart.js doughnut chart to show cost breakdown
  • Add login/register pages connected to JWT auth
‚úÖ OUTPUT: Fully working website ‚Äî users can search, book, and calculate trip cost
06
Testing + Bug Fixing
Days 11‚Äì12 ¬∑ 16h
  • Write JUnit 5 tests for all services (ExpenseCalculatorService, HotelService)
  • Use Mockito to mock repositories in tests
  • Test all API endpoints with Postman collections
  • Fix validation errors and edge cases
  • Finalize Dockerfile + docker-compose.yml for full stack
  • Run: docker-compose up ‚Äî verify full system starts clean
‚úÖ OUTPUT: MVP ready ‚Äî tested, Dockerized, deployable
+
Phase 2: React/Next.js Frontend (after MVP)
+10 days
  • Initialize Next.js project in frontend/ folder
  • Migrate all pages to React components
